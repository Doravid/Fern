So, this was my first in person Computer Science class, second overall Computer Science class at WPI.
This class is still probably the class that taught me the most new information even though I'm a junior now.

The class was structured as so:

- Lecture at **8am** Tuesdays and Fridays
- 10 minute quiz every Friday.
- 4 Major Assignments, 1 Minor Assignment

Overall I think this structure worked pretty well. You kind of perpetually had to study, either for the quizzes or doing the homework.
The first assignment was just setting up a Linux VM with Virtual Box. Virtual Box ended up being really laggy, hard to copy/paste, and hard to submit assignments on.
To save my self from misery, I set up the VM to be headless and then SSHed into the VM with VS Code. This already made my life a lot better,
however I ended up having some more problems with the VM that I will get into later.

The first **real** assignment was actually really cool. It was to create a simple shell **lsh**. Think like `sh` or `bash`.
We had to be able to handle a few intrinsics like cd, ls, etc and then also had to handle piping data between programs.
**Example:** `cat example.txt | grep red`

This assignment shows why I like this class. The projects are mostly about doing cool stuff. A lot of classes teach cool stuff but this one has some of the most building cool stuff of any I have taken. 

The second assignment is my favorite assignment of the class, any maybe ever? It's to take a CPU raytracer, and to make it faster by using threads. 

It's a novel idea, cool context, good use case (but also a pain to grade!). I spent a lot of time on this assignment just trying to get it to go faster, even after I technically finished the assignment. Super fun. The main speed up came from launching some number of threads, say 4, and then splitting the frame you need to render into four even chunks and having each thread render a chunk. Pretty simple! 

However the one problem I had with this assignment led to a **LOT** of lasting effects. For a very long time, I could not get it to speed up no matter what I did. I tried everything I could to get it to run faster, but at most it would run maybe 20% faster. No where near the expected 2-4x speed increase. Well it turns out the VM just didn't work properly and could not use more than one thread.

So I did what any sane person would do: Install Linux on my PC. I could make a whole article on this, but to keep it brief: I installed Pop!_OS (I thought it was funny because Linus from the YT channel LTT bricked his install), and did all my subsequent homework on Linux... and then also my classes next term... and then I bricked my Windows 11 install... and now I've been using Linux for about a year! 

The third assignment was littery the worst thing I've ever seen. It was to create a memory manager. This means you had to implment an API so a program could request memory, write to bytes in memory, read from a byte in memory. It 
was a cool assignment but for some reason, once I got to the part where you have to implement swap, writing memory to disk because you ran out of physical memory, it become nearly impossible. 

You see the issue was as such: A program requests a page. Great, Cool, Fantastic. However, physical memory is full, so, you need to write a page to disk. However, what happens if the processes page table (the meta data for the process) is already written to disk? I had no idea. I `bash`ed my head against the wall for a while until I realized a solution that made me feel stupid for being stuck: Just write the page to disk like normal...

Then read the page table for that page into the newly freed memory, update it to say the page we just swapped out is invalid, then write it back to disk. 

The last assignments was honestly really easy, it was something along the lines of write a few schedulers and test them. Like Round-Robin (1 -> 2 -> 3 -> 1 -> 2 -> 3), FIFO (First In First Out), FILO (First In Last Out)

This was the most boring assignment, but also, I was pretty burnt out from project three, so I appreciated the layup and easy points for my grade. It made the last quiz less stressful since I knew this assignment would bring up my grade.

## Conclusion

This was a super fun class! I recommend everyone to take it! I would say make sure your solid on your C programming concepts as the class didn't really give you time to learn the language if you didn't already know it.
